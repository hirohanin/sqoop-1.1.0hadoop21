Release Notes for Sqoop 1.1.0: September, 2010

This document lists all Sqoop issues included in version 1.1.0 not present in the previous release, 1.0.0.

Bug fixes:

  • [SQOOP-62] - Fix failing Oracle compatibility tests.
  • [SQOOP-94] - COMPILING.txt does not mention all dependencies.
  • [SQOOP-86] - Release note generation requires python 2.5.
  • [SQOOP-85] - Additional documentation build adjustments.
  • [SQOOP-81] - Creation of /tmp/sqoop/compile is not multi-user friendly.
  • [SQOOP-84] - Fix documentation build error under CentOS.
  • [SQOOP-80] - bin/configure-sqoop improperly escapes shell variable names.
  • [SQOOP-47] - NPE if --append is specified with HBase import target.
  • [SQOOP-72] - sqoop-eval should use PreparedStatement.execute().
  • [SQOOP-38] - Direct PostgreSQL import only works when providing a port.
  • [SQOOP-68] - Delimiters in strings are not escaped properly.
  • [SQOOP-66] - Jar for ConnManager class is not included in job payload.
  • [SQOOP-63] - JobBase classes unnecessarily create ConnManagers.
  • [SQOOP-52] - Stop TestHBaseImport from timing out.
  • [SQOOP-55] - BaseSqoopTestCase.createTableWithColTypes() calls close() twice.
  • [SQOOP-45] - If compilation dir is unspecified, use a nonce directory.
  • [SQOOP-6] - Job completion percentage is inaccurate.
  • [SQOOP-15] - Do not assume that Job.getCounters() returns non-null.
  • [SQOOP-32] - sqoop-export documentation in SqoopUserGuide refers to 'import-args'.
  • [SQOOP-22] - Sqoop date tests are too permissive.
  • [SQOOP-2] - MySQLExportMapper does not send username.

Improvements:

  • [SQOOP-89] - Support multiple output files in ManagerCompatTestCase.
  • [SQOOP-88] - Parameterize pre-table-import validity checks in SqlManager.
  • [SQOOP-79] - Add scripts to start/stop metastore in background, use pidfiles.
  • [SQOOP-51] - Document driver jar installation process.
  • [SQOOP-75] - Allow configuration of ManagerFactories through a configuration subdirectory.
  • [SQOOP-73] - Generated records should handle more boolean value formats.
  • [SQOOP-40] - SqoopOptions should use annotations to bind fields to properties.
  • [SQOOP-69] - Improve data accessible to ManagerFactory.accept().
  • [SQOOP-65] - Refactor some utilities out of MySQLExportMapper.
  • [SQOOP-59] - 'sqoop eval' should support DDL/DML operations.
  • [SQOOP-56] - Improve parameterization of ManagerCompatTestCase.
  • [SQOOP-54] - Allow per-manager configuration of isolation level for metadata queries.
  • [SQOOP-53] - Build test jar and publish to maven.
  • [SQOOP-35] - Bundle job dependencies via DistributedCache.
  • [SQOOP-46] - Sqoop 'verbose' mode shows too much IPC chatter.
  • [SQOOP-10] - Provide a useful error message when a Java compiler cannot be found.
  • [SQOOP-4] - Add update-mode export process.

New features:

  • [SQOOP-90] - Tool to merge datasets imported via incremental import.
  • [SQOOP-76] - Add automated release note generation script.
  • [SQOOP-36] - Redistribute mysql-connector-j with Sqoop.
  • [SQOOP-41] - Add support for incremental imports.
  • [SQOOP-39] - (SIP-6) Session Management API.
  • [SQOOP-14] - Allow Sqoop to import data into HBase.
  • [SQOOP-3] - Add ability to import arbitrary queries.
  • [SQOOP-16] - Generate Eclipse project with ant.
